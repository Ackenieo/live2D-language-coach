# Live2D 口语陪练 H5 平台 — 产品设计文档

## 1. 产品概述

### 1.1 产品定位
基于 Live2D 虚拟角色的 AI 英语口语陪练 H5 移动端应用。用户通过与 Live2D 角色的实时语音对话，在角色扮演场景中练习口语，获得多维度评分反馈。

### 1.2 目标用户
- 英语口语学习者（初级～高级）
- 需要模拟真实对话场景的备考用户
- 对虚拟角色互动感兴趣的年轻用户

### 1.3 核心卖点
- Live2D 角色实时表情/嘴型/动作反馈
- AI 语音对话，沉浸式角色扮演场景
- 多维度口语评分 + 个性化改进建议
- 黑灰高级配色，极简科技感 UI

**交互逻辑**：
1. 输入手机号 → 校验格式（11位大陆手机号）
2. 点击「发送验证码」→ 按钮 60 秒倒计时
3. 输入 6 位验证码 → 点击「登录」
4. 后端校验验证码，首次登录自动创建用户
5. 返回 JWT token，跳转欢迎页

**接口**：
- `POST /api/auth/send-sms` — 发送短信验证码
- `POST /api/auth/login` — 验证码登录
- `POST /api/auth/refresh` — 刷新 token

### 3.3 对话页面 (`/chat`)

**页面布局**（核心页面）：

```
┌────────────────────────────────┐
│  ← 结束对话    ⏱ 03:24    📷  │  ← 顶栏
├────────────────────────────────┤
│                                │
│   ┌──用户气泡──────────────┐   │
│   │ I'd like to check in.  │   │
│   └────────────────────────┘   │
│                                │
│      ┌─────────────────┐      │
│      │                 │      │
│      │   Live2D 角色    │      │  ← 居中 Live2D
│      │   (唇动+表情)    │      │
│      │                 │      │
│      └─────────────────┘      │
│                                │
│   ┌──AI气泡───────────────┐   │
│   │ Certainly! Do you     │   │
│   │ have a reservation?   │   │
│   └────────────────────────┘   │
│                                │
├────────────────────────────────┤
│  场景: 酒店入住 | Easy | 美式    │  ← 场景信息栏
├────────────────────────────────┤
│  🎤 按住说话          ⌨️ 文字  │  ← 输入栏
└────────────────────────────────┘
```

**核心功能**：

#### 3.3.1 Live2D 角色展示区（居中）
- Live2D 模型渲染，占据画面中央约 40% 高度
- **表情反馈**：根据对话情绪展示开心/思考/鼓励等表情
- **嘴型同步**：AI 语音播放时角色嘴型同步动画
- **待机动作**：无对话时播放呼吸/眨眼/微动待机动画
- **反馈动作**：用户得分高时播放鼓励动作

#### 3.3.2 毛玻璃对话框
- **用户气泡**（右侧）：深灰毛玻璃 `rgba(26,26,26,0.75)` + 模糊 + 右侧对齐
- **AI 气泡**（左侧）：深灰毛玻璃 `rgba(26,26,26,0.75)` + 模糊 + 左侧对齐 + 青绿左边框
- 气泡内显示文字内容
- 自动滚动到最新消息
- 支持同时显示多轮对话（最多可见 3 轮，超出滚动）

#### 3.3.3 计时器
- 顶栏居中位置，格式 `⏱ MM:SS`
- 进入对话即开始计时
- 结束对话时记录总时长

#### 3.3.4 摄像头识别栏目
- 顶栏右侧 📷 图标按钮
- **仅开关控制，不显示摄像头视频画面**
- 开启后浏览器获取摄像头权限
- 用途：
  - 检测用户是否在屏幕前（专注度）
  - 面部表情分析（微笑/困惑/思考）
  - 检测开口说话状态
- 关闭时仅使用麦克风进行语音对话

#### 3.3.5 实时综合得分
- 顶栏左下方或场景信息栏内，显示当前得分
- 格式：`Score: B`（大写字母）
- 实时更新（每轮对话后刷新）
- 字母等级：**S / A / B / C / D / E**

#### 3.3.6 内置切换入口（对话页内可切换）
- 场景信息栏点击可弹出切换面板：
  - 角色扮演场景切换（弹窗选择）
  - 难度切换（Easy / Medium / Hard）
  - 语音口音切换（美式 / 英式）
- 场景和难度即时切换，下次对话生效

#### 3.3.7 输入方式
- **语音输入**（主要）：按住录音按钮说话，松手发送
- **文字输入**（辅助）：点击键盘图标切换到文字输入模式
- 录音时按钮有波纹动画反馈
- ASR（自动语音识别）将语音转为文字后展示在气泡中

**评分维度说明**：
- **准确度 (Accuracy)**：发音准确性、语法正确性

- **完整度 (Completeness)**：回答是否完成对话目标、信息量是否充足
- 每项评分均为字母等级：**S / A / B / C / D / E**

**建议生成**：
- AI 根据对话内容自动生成 3~5 条个性化改进建议
- 涵盖发音、词汇、语法、流利度等方面

## 7. 技术栈建议

### 7.1 前端
- **框架**: Vue 3 + TypeScript + Vite
- **UI 框架**: 自定义（黑灰主题）或基于 Tailwind CSS
- **Live2D 渲染**: Live2D Cubism SDK for Web
- **语音**: Web Speech API / WebRTC（录音）+ 后端 TTS 返回音频
- **摄像头**: MediaDevices getUserMedia（仅检测，不显示画面）
- **状态管理**: Pinia
- **路由**: Vue Router
- **HTTP 客户端**: Axios

### 7.2 后端（基于现有架构扩展）
- Spring Boot 3.5 + MyBatis-Plus + Redis + MySQL
- AI 对话：接入 LLM API（如 OpenAI / Claude）
- ASR（语音识别）：接入第三方 ASR 服务
- TTS（语音合成）：接入第三方 TTS 服务
- 文件存储：OSS（头像上传）

## 8. 非功能需求

- 语音响应延迟 < 3 秒
- Live2D 渲染帧率 ≥ 30fps
- H5 适配 375px ~ 428px 移动端宽度
- 支持 PWA 离线缓存（Live2D 模型资源）
- 网络异常时友好提示 + 自动重连

## 10. 开发阶段建议

| 阶段 | 内容 | 说明 |
|------|------|------|
| Phase 1 | 基础框架 + 登录 | 复用现有登录模块，搭建前端壳 |

| Phase 4 | Live2D 集成 | 加载模型 + 基础动画 |
| Phase 5 | AI 对话对接 | LLM + TTS 集成 |
| Phase 6 | 语音输入 | ASR 录音 + 识别 |
| Phase 7 | 评分报告 | 评分算法 + 报告页面 |
| Phase 8 | 排行榜 + 个人中心 | 数据展示 + 用户管理 |
| Phase 9 | 摄像头检测 | 面部检测（无画面） |
| Phase 10 | 优化打磨 | 性能 + 动画 + 异常处理 |
